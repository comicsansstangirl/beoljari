# 기획 02 · 학년이 바뀌어도 이어지는 교실 「별자리」

> **한 줄 정의**
> 아이들의 성향을 입력하면 짝꿍·모둠을 자동 배치하고, 그 성향 데이터를 학년이 올라가도 **다음 담임에게 인수인계**하는 교실 운영 웹.

---

## 과제 조건 매핑 (애프터 제안 프레임 01~05)

**01. 우리가 만난 현장**
3월 첫 주, 담임이 이름표 30장을 책상에 늘어놓고 밤늦게까지 자리 배치를 짜는 장면.
"얘랑 얘는 붙이면 안 되고, 얘는 앞자리, 이 모둠은 너무 조용하고…"
그 장면 속 진짜 기대 → 교사는 예쁜 자리표가 아니라 **아이들이 안전하게 섞이고 잘 배우는 교실**을 만들고 싶었다.

**02. 비상이 지금 답하는 방식**
비상은 **콘텐츠(교재·강의·평가)**로 교사를 돕는다. "무엇을 가르칠까"엔 강하다.

**03. 아직 닿지 않은 지점**
교사의 진짜 노동은 콘텐츠가 아니라 **30명을 어떻게 배치하고 관계를 관리하느냐**다.
그런데 "누가 누구랑 맞는지, 누굴 떼어놔야 하는지"라는 값진 지식은 **오직 담임 머릿속에만** 있다.
학년이 올라가면 이 지식은 통째로 리셋되고, 새 담임은 3월에 또 맨땅에서 시작한다.
이 "교실 운영·관계 데이터"는 지금 어디에도 남지 않고, 고객(교사)에게도 비상에게도 **보이지 않는다.**

**04. 달라진 현장, 더 필요한 역할**
비상이 답할 역할은 "콘텐츠 공급자"를 넘어 **교실 운영 파트너**다.
교사의 암묵지(누가 누구랑 맞는지)를 데이터로 쌓아, *학년을 넘어 이어지는 자산*으로 만드는 역할.

**05. 신뢰를 쌓는 다음 장면 (애프터)**
- **누구:** 4학년 새 담임 / 그리고 지난해 이 아이들을 봤던 3학년 담임
- **이전:** 새 담임은 아이들을 모른다. 자리 배치는 감으로, 사고는 터진 뒤에 안다. 3학년 담임의 1년치 관찰은 종업식과 함께 증발한다.
- **이후:** 새 담임이 로그인하면 **작년 담임이 남긴 성향 데이터**가 떠 있다. "학습 분위기 안정" 목적으로 배치안 3개가 10분 만에 나온다. 첫날부터 "이 둘은 떼어놓는 게 낫겠다"를 *이미 알고* 시작한다.

---

## 문제정의

1. **감정노동·시간의 블랙박스** — 자리·모둠 배치는 매달 반복되는 고난도 작업인데, 전부 교사 머릿속 계산에 의존한다. 명시적 도구가 없다.
2. **지식의 휘발** — "누가 누구랑 갈등, 누가 분위기 메이커"라는 1년치 관찰이 학년 진급과 함께 사라진다. 다음 담임은 매번 처음부터.
3. **목적 없는 배치** — 지금 배치는 "섞기" 수준에 머문다. *학습 분위기 안정 / 활발한 토론 / 관계 회복* 같은 목적별 최적화가 안 된다.

## 타겟

| 구분 | 대상 | 왜 |
|---|---|---|
| 핵심 사용자 | 초·중등 담임 교사 | 배치·모둠 편성의 실제 주체 |
| 데이터 지속 주체 | 학교(학년 인수인계) | 성향 데이터가 학년을 넘어 이어져야 가치 발생 |
| 확장 이해관계자 | 학년부·상담교사 | 관계 데이터를 생활지도·상담에 재활용 |

> 핵심은 **1회성 배치 툴이 아니라 "이어지는 데이터"**라는 점. 여기가 경쟁 무료 자리배치 앱과 갈리는 지점.

## 기능정의

**A. 성향 입력 — 4축 성향 카드 (MBTI형)**
각 학생은 4개 축의 양극 사이에서 선택(3~5점 슬라이더). 4글자 성향 코드로 요약 표시 → 한눈에 읽힘.

| 축 | 한쪽 극 | 반대쪽 극 | 배치에 쓰이는 방식 |
|---|---|---|---|
| 에너지 | 조용(Q) | 활발(L) | 활발형이 한 구역에 몰리지 않게 분산 |
| 주도성 | 주도(D) | 수용(F) | 모둠마다 주도형 최소 1명 보장 |
| 집중 | 산만(S) | 몰입(C) | 산만형은 몰입형·조용형 곁에 앵커링 |
| 협동 | 개별(I) | 협동(T) | 협동형을 고립 학생 옆에 배치 |

> 예: `L·D·C·T` = 활발·주도·몰입·협동형 (분위기 메이커).
> 5번째 축(감정 예민도 등)은 필요 시 확장. 우선 4축으로 시작.

**A-2. 관계 데이터 — 연결 태그**
학생 카드에 "연결된 학생"을 태그로 저장. 클릭하면 해당 학생 정보 카드로 이동.
- `[시너지]` — 함께 두면 서로 좋은 영향 (긍정)
- `[주의]` — 함께 두면 마찰·산만 유발 → 떼어놓기 (부정) *※ 라벨은 `주의`/`거리두기` 중 택1, 낙인 피하려 순한 단어 사용*

```
김학생 님과 연결된 학생
  [시너지]  @김반장   @이차분
  [주의]    @박세모
```
→ `@김반장` 누르면 김반장 성향·연결 카드가 뜸 (관계망 탐색 가능).

**B. 목적 기반 자동 배치 (핵심) — 3요소 종합 계산**
배치는 **① 성향 4축 · ② 관계 태그 · ③ 배치 목적** 세 가지를 함께 넣어 점수화한다. 목적에 따라 각 요소의 가중치가 달라짐.

| 배치 목적 | 정의(무엇을 최적화) | 성향 규칙 | 관계 규칙 |
|---|---|---|---|
| **수업 안정** | 수업 방해 최소화 | 활발·산만·주도 성향이 한 구역에 몰리지 않게 분산 | `[주의]` 짝 **강하게 분리** |
| **모둠 토론 활성** | 모든 모둠에서 대화가 끊기지 않게 | 모둠마다 주도형 ≥1 + 수용/협동형 섞기 | `[시너지]` **최대화** |
| **관계 회복** | 소외·마찰 최소화 | 협동형을 고립 학생 옆에 배치 | 연결 0개 학생 우선 케어 + `[주의]` 분리 |
| **다양성 섞기** | 끼리끼리 방지 | 모둠 내 성향 축이 한쪽으로 쏠리지 않게(분산 최대) | 기존 `[주의]`만 회피 |

- 계산 흐름: 후보 배치 생성 → (성향 점수 + 관계 점수) × 목적 가중치로 스코어링 → 상위 **배치안 3개** + 각 안의 근거 설명 자동 생성("이 모둠은 주도형 1 + 수용형 3, [주의] 짝 분리 완료").
- 교사가 하나 골라 드래그로 미세조정 → 자리표 확정·출력.
- 하드 제약(반드시 지킴): `[주의]` 짝 같은 짝꿍 금지 / 주도형 0명 모둠 금지.

**C. 학년 인수인계 (차별점)**
- 성향·관계 데이터를 DB에 저장 → 학년 진급 시 다음 담임에게 이관
- 새 담임은 **읽기 권한으로 시작 → 관찰 후 업데이트·수정**
- 변경 이력 남김(누가 언제 무엇을 바꿨는지) → 신뢰·책임 구조

**D. 프라이버시·윤리 설계** *(민감 데이터라 반드시)*
- 성향·관계 데이터는 **교육 목적 한정**, 낙인 방지를 위해 "판정"이 아니라 "관찰 메모" 톤으로 저장
- 학부모 열람 범위·삭제권 정책 명시 (비상의 신뢰 자산과 연결)
- 아이 본인에게 불리하게 쓰이지 않도록 접근 권한을 담임·학년부로 제한

## 해피패스

```
1. 3월, 4학년 새 담임 로그인
2. 화면에 작년(3학년) 담임이 남긴 성향·관계 데이터가 이미 떠 있음
3. 배치 목적 → "수업 안정" 선택
4. 성향 4축 + 관계 태그 + 목적을 종합 계산 → 배치안 3개 생성 (각 안에 근거 설명 포함)
5. 담임이 2안 선택 → 창가 자리 하나만 드래그로 조정
6. 자리표 확정 → PDF 출력, 교실에 게시
   (예전 2시간 → 10분)
7. 학기 중 새로 관찰한 내용 업데이트
8. 종업 시 → 데이터가 다시 다음 담임에게 이관
```

## 프로토타입 범위 (데모에서 실제 만들 것)

- **필수:** 4축 성향 입력 + 연결 태그(`[시너지]`/`[주의]`) → "목적 선택" → **배치안 자동 생성**되는 웹 화면 1개. 버튼 누르면 성향·관계·목적 3요소로 실제 계산돼서 나오게(규칙 기반 목업이라도 실동작하면 임팩트 큼). 태그 학생 클릭 시 정보 카드 뜨는 인터랙션 포함.
- **핵심 데모 컷:** "새 담임이 로그인하니 작년 데이터가 이미 있다" 장면 → 과제의 "누구의 경험이 어떻게 달라지는지"를 가장 강하게 보여주는 화면. **이 인수인계 장면이 발표의 킬러 컷.**
- **선택:** 배치 근거 설명 툴팁, 변경 이력 화면.

## 참신성·현장성 요약

- **참신성:** 자리배치를 "1회성 잡무 툴"이 아니라 *학년을 넘어 축적되는 교실 데이터 자산*으로 재정의. 비상을 콘텐츠 회사에서 **교실 운영 파트너**로 확장.
- **현장성:** 교사의 가장 반복적이고 안 보이던 고통(관계·배치)에 정확히 꽂힘. 비바샘 교사망 위에 자연스럽게 얹힘.
